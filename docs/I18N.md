# Multi-language support (i18n)

The app uses [easy_localization](https://pub.dev/packages/easy_localization) for Turkish (tr-TR) and English (en-US).

## Setup

- **Translation files**: `assets/translations/tr-TR.json` and `assets/translations/en-US.json`
- **Locale helper**: `lib/core/locale_helper.dart` — loads saved locale from `shared_preferences` or detects system language on first launch

## Auto-detect on first launch

- If the user has **never** chosen a language, the app uses the device’s system locale.
- If the system language is **Turkish** (`tr`), the app uses **tr-TR**.
- Otherwise it defaults to **en-US**.

## Persistence

- The selected language is stored under the key `app_locale` in `SharedPreferences` (e.g. `tr_TR`, `en_US`).
- On the next launch, `LocaleHelper.getStartLocale()` reads this value and the app starts in the saved language.

## Changing language (Settings)

- **Driver app**: Profile → **Ayarlar** (Settings) → **Dil** (Language).
- A bottom sheet offers **Türkçe** and **English**. Choosing an option saves the locale and calls `context.setLocale(...)` so the UI updates immediately.

## Using translations in code

1. Import: `import 'package:easy_localization/easy_localization.dart';`
2. Use: `'key'.tr()` for a string from the JSON files.

Example: `Text('request_tow'.tr())`

## Adding new strings

1. Add the key and value to both `assets/translations/tr-TR.json` and `assets/translations/en-US.json`.
2. Use `'your_key'.tr()` in the Dart code.

## Main entry (main.dart)

- `EasyLocalization.ensureInitialized()` is called at startup.
- `LocaleHelper.getStartLocale()` provides the initial locale (saved or auto-detect).
- The app is wrapped in `EasyLocalization` with `path: 'assets/translations'`, `supportedLocales: [tr, en]`, and `fallbackLocale: en_US`.
- `MaterialApp` uses `context.localizationDelegates`, `context.supportedLocales`, and `context.locale` so that `tr()` resolves correctly everywhere.
